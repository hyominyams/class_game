# 🗺️ ClassQuest Frontend Implementation Rules

이 문서는 **ClassQuest**의 프론트엔드 개발을 위한 상세 구현 규칙입니다. `PRD.md`와 `Agents.md`를 기반으로 하며, Supabase 연동 전 **UI/UX 완성**을 최우선 목표로 합니다.

## 1. 🎯 개발 원칙 (Development Principles)

1.  **Frontend First**: 백엔드 로직 없이도 완전한 UX를 경험할 수 있도록 구현합니다.
    *   모든 데이터는 `mock` 데이터를 사용하여 하드코딩하거나 상수로 관리합니다.
    *   API 호출 로직은 추후 `Server Actions`로 대체하기 쉽도록 인터페이스만 정의합니다.
2.  **Pixel Art System**: `.agent/rules/design-system-rules.md`를 엄격히 준수합니다.
    *   모든 그릇(Container)은 `border-4 border-black`과 `box-shadow`를 가집니다.
    *   폰트는 픽셀 폰트와 고딕을 적절히 썩어 사용합니다.
3.  **Role-Based Routing**:
    *   학생: `/student/*`
    *   교사: `/teacher/*`
    *   관리자: `/admin/*`
4.  **Tablet Optimized**: **태블릿 PC 가로 버전 (Landscape)**을 기준 해상도로 하여 레이아웃을 구성합니다. 모든 터치 영역은 충분히 크게 확보합니다.

---

## 2. 🧑‍🎓 학생 (Student) 구현 규칙

학생은 **"몰입감과 성장"**이 핵심입니다. 게임과 보상에 집중된 UI를 구성합니다.

### 2.0 최초 로그인 (Onboarding)
*   **온보딩 모달**: 초기 비밀번호(`a123456789`)로 첫 로그인 시 자동 팝업.
    1.  **닉네임(별명) 입력**: 개성 있는 픽셀 닉네임 설정.
    2.  **비밀번호 변경**: 새로운 암호 입력 및 확인.
    *   완료 전까지는 대시보드 진입 차단.

### 2.1 대시보드 (`/student/dashboard`)
*   **상태 요약 카드**:
    *   **내 코인**: 픽셀 동전 아이콘 + 큰 숫자 표기.
    *   **현재 랭킹**: 학급 내 등수 및 티어(브론즈/실버/골드) 뱃지 표시.
    *   **월간 평균 점수**: 막대 그래프 또는 꺾은선 그래프로 시각화.
*   **바로가기 메뉴**: '게임 하러 가기', '상점', '내 기록' 등으로 이동하는 큰 버튼.

### 2.2 게임 목록 및 플레이 (`/student/game`)
*   **게임 선택 화면**:
    *   썸네일이 포함된 게임 카드 리스트 (사칙연산, 영어단어, 사회 퀴즈, 픽셀 슈팅 쇼케이스).
    *   카드에는 '오늘 남은 코인 획득 횟수(0/3)' 뱃지 표시.
*   **게임 모드 선택 모달**:
    *   **연습 모드**: "무제한 플레이 / 코인 획득 가능" 설명.
    *   **대회 모드**: "학급/학년 대회 진행 중"일 때만 활성화. "남은 기회: 3회" 강조 표시.
*   **게임 플레이 화면 (`/student/game/[id]`)**:
    *   중앙 `iframe` 또는 Canvas 영역 (Phaser 연동용, 현재는 Placeholder).
    *   좌우 측면에 현재 점수 및 남은 시간 표시 UI 배치.

### 2.3 상점 (`/student/store`)
*   **아이템 리스트**: 쿠폰 등 구매 가능한 아이템을 그리드 형태로 나열.
*   **구매 인터랙션**: 구매 버튼 클릭 시 코인 차감 애니메이션 및 '구매 완료' 픽셀 모달 팝업.

### 2.4 통계 (`/student/stats`)
*   **문제 복습**: 최근 틀린 문제 리스트업. (클릭 시 정답/해설 보기 아코디언)
*   **대회 기록**: 과거 참여한 대회의 순위와 점수 히스토리.

## 3. 👨‍🏫 교사 (Teacher) 구현 규칙

교사는 **"관리와 모니터링"**이 핵심입니다. 한눈에 학급 상황을 파악해야 합니다.

### 3.1 대시보드 (`/teacher/dashboard`)
*   **학급 현황**:
    *   총 학생 수, 현재 접속 중인 학생 수(Mocking).
    *   오늘의 평균 점수 및 코인 획득 총량 요약.
*   **빠른 작업**: '대회 열기', '코인 보너스 주기' 퀵 버튼.

### 3.2 학생 관리 (`/teacher/students`)
*   **학생 리스트 테이블**:
    *   컬럼: 이름, 학번, 현재 코인, 최근 접속일.
    *   **액션**: '코인 지급' 버튼 (개별 지급 모달 팝업), '비밀번호 초기화' 버튼.
*   **계정 생성**: '학생 추가' 버튼 -> 이름, 번호 입력하여 계정 생성 (초기 비번 자동 설정).
*   **일괄 지급 기능**: 체크박스로 여러 명 선택 후 '보너스 코인 일괄 지급' 버튼.

### 3.3 문제 세트 관리 (`/teacher/questions`)
*   **세트 리스트**:
    *   상태 뱃지: `ACTIVE`(녹색), `INACTIVE`(회색).
    *   각 카드에 '복제', '수정', '삭제' 버튼 포함.
*   **에디터 UI**:
    *   좌측: 문항 리스트 (Drag & Drop으로 순서 변경 가능 UI).
    *   우측: 문제 입력 폼 (지문, 보기, 정답, 해설).
    *   제약 사항: 최대 34문제 카운터 표시.

### 3.4 대회 운영 (`/teacher/tournaments`)
*   **대회 생성 위자드**:
    *   대회명, 기간(시작~종료), 대상 게임 선택 단계.
*   **진행 중인 대회**:
    *   실시간 리더보드 (자동 새로고침 UI).
    *   '대회 종료' 비상 버튼 (빨간색 픽셀 버튼).

## 4. 🛠️ 관리자 (Admin) 구현 규칙

관리자는 **"시스템 제어"**가 핵심입니다. 전역 설정과 데이터 관리를 담당합니다.

### 4.1 전역 대시보드 (`/admin/dashboard`)
*   **시스템 지표**: 전체 가입자 수, 오늘 플레이 횟수 등 KPI 카드.
*   **로그 뷰어**: 최근 시스템 로그(가입, 오류, 결제 등)를 터미널 스타일로 표시.

### 4.2 계정 관리 (`/admin/users`)
*   **계정 생성 폼**:
    *   탭 구분: [교사 관리] / [학생 관리]
    *   **교사 관리**: 교사 계정 생성 (이름, 담당 학급/반, ID).
    *   **학생 관리**: 전교생 조회 및 관리 (학년/반 필터링).
*   **비밀번호 초기화**: 특정 계정 선택 후 '비밀번호 리셋' 버튼 (초기값: `a123456789` 안내 팝업).
*   **전체 코인 지급**: 특정 학년 또는 전교생 대상 코인 일괄 지급 기능.

### 4.3 학년 대회 관리 (`/admin/grade-tournaments`)
*   교사의 대회 관리와 유사하되, 스코프가 `GRADE` 단위로 설정됨.
*   학년 전체 랭킹 뷰 제공.

## 5. 🎨 공통 컴포넌트 요구사항 (Component Requirements)

다음 컴포넌트들은 `components/ui`에 우선적으로 구현되어야 합니다.

1.  **`PixelCard`**: 흰 배경, 검은 테두리, 그림자가 적용된 기본 컨테이너.
2.  **`PixelButton`**: 눌리는 효과가 있는 3D 픽셀 버튼 (Primary, Danger, Outline 변형).
3.  **`PixelBadge`**: 랭킹, 상태(Active/Inactive) 등을 표시하는 작은 라벨.
4.  **`PixelInput`**: 각진 테두리의 입력 필드.
5.  **`PixelModal`**: 화면 중앙에 뜨는 알림/입력 창 (검은 배경 오버레이 포함).
6.  **`PixelTabs`**: 폴더 탭 스타일의 네비게이션 컴포넌트.

## 6. 개발 단계 (Milestones)

1.  **Step 1: 공통 컴포넌트 (Design System)**
    *   버튼, 카드, 입력창, 모달 등 기본 빌딩 블록 완성.
2.  **Step 2: 학생 뷰 (Student View)**
    *   대시보드 -> 게임 목록 -> 게임 플레이(껍데기) 순으로 구현.
3.  **Step 3: 교사 뷰 (Teacher View)**
    *   대시보드 -> 학생 관리 -> 문제 세트 관리 순으로 구현.
4.  **Step 4: 관리자 뷰 (Admin View)**
    *   계정 생성 및 전역 설정 페이지 구현.
