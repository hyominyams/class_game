# PRD: 학급 게임 학습 플랫폼 **ClassQuest**

## 1. 프로젝트 개요

### 1.1 목적

ClassQuest는 여러 개의 학습 게임을 하나의 플랫폼에서 운영하며,
**게임 결과를 통합 집계하여 랭킹·대회·보상 시스템으로 연결**하는 학급용 게임 학습 플랫폼이다.

* 학생: 게임을 통한 몰입형 학습 + 명확한 경쟁 구조
* 교사: 학급 단위 운영·대회 관리·학습 데이터 분석
* 관리자: 전역 설정 및 학년 단위 이벤트 운영

### 1.2 핵심 컨셉

> **연습은 자유롭게, 경쟁은 공정하게, 결과는 하나로**

* 게임은 분산되어 실행
* 결과는 중앙에서 통합 관리
* 리그·승급은 “대회 모드”만으로 결정

### 1.3 디자인 및 시각적 정체성

* **테마**: **Pixel Art UI (정교한 고전 게임 및 PC OS 감성)**
* **핵심 컨셉**: 어린이들에게 친숙한 아기자기함 + 정교한 8비트 픽셀 디테일
* **주요 요소**: 연한 미색 배경, 굵은 검은색 테두리, 입체적인 오프셋 그림자, 픽셀 폰트
* **최적화 기기**: **태블릿 PC (가로 모드)** 최우선 최적화
* **색상 팔레트**: `#fdf5e6` (배경), `#ff2e63` (강조 핑크), `#08d9d6` (강조 민트), `#18181b` (라인)

## 2. 사용자 유형 및 권한

### 2.1 계정 권한 구조

| 역할          | 설명                        |
| ----------- | ------------------------- |
| **Teacher** | 학급 단위 운영자 · **본인 학급(학년, 반) 학생 계정만 생성 가능** |
| **Student** | 게임 플레이어                   |

### 2.2 공통 속성

* 모든 계정은 **학년, 반**을 반드시 가짐
* Admin이 계정 생성 및 권한 부여

### 2.3 권한별 기능

#### Admin

* **서비스 총괄 관리 (Not 서버 관리자)**
* 전역 설정 관리
* GLOBAL 문제 세트 관리
* 모든 학급/학년 문제 세트 수정 가능
* 학급 대회 + **학년 대회** 생성/운영
* 전체 통계/감사 로그 확인

#### Teacher

* **자기 학급(동일 학년, 반) 학생 계정 생성 및 조회**
* 학급 문제 세트 관리
* 학급 대회 생성/시작/종료
* 학급 학생에게 코인 보너스 지급
* 학급 대시보드 확인

#### Student

* 게임 플레이
* 개인 대시보드 확인
* 랭킹/대회 결과 확인
* 코인 사용(상점)

## 3. 인증 및 계정 관리

### 3.1 계정 생성

* **Admin**: 교사 및 전교생 계정 생성 가능
* **Teacher**: 본인 학급(학년, 반이 동일한 경우)의 학생 계정 생성 가능
* **초기 비밀번호**: `a123456789`
* **최초 로그인 필수 작업**: 
    1. **자신의 별명(Nickname)** 설정
    2. **새로운 암호** 설정
* DB에는 **해시값만 저장** (비밀번호)

### 3.2 로그인

* 방식: 학번(ID) + 비밀번호
* DB 기반 검증
* HttpOnly 쿠키 세션 사용

### 3.3 비밀번호 관리

* 분실 시: Admin이 초기 비밀번호로 리셋
* `must_change_password` 플래그로 변경 강제

## 4. 게임 시스템

### 4.1 게임 개념

* 게임 엔티티 명: **`class_game`**
* 게임 수는 고정되지 않음 (확장 가능)
* **주요 게임 카테고리**: 사칙연산, 영어단어, 사회 퀴즈, 픽셀 슈팅 등
* 모든 게임은 공통 점수/결과 구조 사용

### 4.2 게임 모드

#### 일반 모드

* 플레이 횟수: **무제한**
* 점수/정답률: **매판 DB 저장**
* 코인 지급: **게임별 하루 최대 3회**
* 목적: 연습 + 코인 수급 + 학습 데이터 축적

#### 대회 모드

* 게임별 도전 횟수: **3회**
* 초과 시 해당 게임 **잠금**
* 점수 집계: **게임별 최고 점수만 반영**
* 종합 점수 = 각 게임 최고 점수 합산

---

## 5. 대회 시스템

### 5.1 대회 스코프

| 스코프   | 설명                  |
| ----- | ------------------- |
| CLASS | 학급 단위 대회            |
| GRADE | 학년 단위 대회 (Admin 전용) |

### 5.2 대회 규칙

* 대회 기간 동안만 점수 유효
* 일반 모드 플레이는 대회 중에도 허용
* 대회 종료 시 결과 확정

### 5.3 리그/승급

* **CLASS 대회만 적용**
* GRADE 대회는 **랭킹 표시만** 제공
* 리그: 골드 / 실버 / 브론즈
* 승급/강등은 CLASS 대회 결과 기준

## 6. 랭킹 시스템

### 6.1 랭킹 종류

* 일반 모드: 주간 / 월간 (표시용)
* 대회 랭킹:

  * 학급 대회 랭킹
  * 학년 대회 랭킹(대회 기간 한정)

### 6.2 학년 랭킹

* Admin이 학년 대회를 열었을 때만 표시
* 해당 대회 점수 기준
* 리그/승급과 무관

## 7. 문제 관리 시스템

### 7.1 문제 세트 구조

* 게임별 문제는 **문제 세트 단위**로 관리
* **기본 문제 형식**: **4지 선다 객관식**
* `CLASS` (학년 + 반)
* **게임 1:1 대응**: 문제 세트는 특정 게임(`class_game`)에 종속되어 관리됨.
* **유연한 변경**: 교사가 특정 게임에 대해 여러 문제 세트(예: "1주차", "2주차")를 만들어두고 수시로 선택/변경 가능.

### 7.2 문제 세트 규칙

* 학급당/게임당 **최대 34개**
* 한 번에 **활성 문제 세트는 1개**
* 활성 세트는 삭제 불가
* 세트 기능:

  * 추가(Add): 빈 세트
  * 수정
  * 복제
  * 삭제(비활성만)
  * 활성화

### 7.3 문제 적용 우선순위

1. CLASS 활성 문제 세트
2. 없으면 GLOBAL 문제 세트

## 9. 학생 페이지 구조 (Student Pages)

### 9.1 대시보드 (Dashboard)
* 현재 코인 및 통합 요약 정보
* 학급 랭킹 요약 (내 순위, 티어)
* 최근 활동 로그
* 바로가기 (게임, 상점 등)

### 9.2 랭킹 (Ranking)
* **주간 랭킹**: 이번 주 획득 점수 기준
* **월간 랭킹**: 이번 달 누적 점수 기준
* **대회 랭킹**:
    * 학급 대회: 티어 시스템 적용 (다이아몬드, 플래티넘, 골드 등)
    * 학년 대회: 단순 순위 나열

### 9.3 학습 기록 (Stats/Learning Records)
* **모은 뱃지 (Badge Collection)**: 획득한 뱃지 시각화 (잠금/해제 상태 표시)
* 게임별 상세 통계
* 월별 성장 그래프
* 문제 오답 노트
